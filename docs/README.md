# Startups India App Docs

These docs are the working source of truth for future changes and chat handoffs.
When the app changes, update the relevant file in this directory in the same
commit as the code change.

- [HANDOFF.md](HANDOFF.md): start here for a quick future-chat briefing.
- [APP_STATE.md](APP_STATE.md): current product state and major user flows.
- [ARCHITECTURE.md](ARCHITECTURE.md): project structure, routing, state, and feature ownership.
- [DATA_CONTRACTS.md](DATA_CONTRACTS.md): Firestore collections, model fields, and rules notes.
- [DECISIONS_AND_RISKS.md](DECISIONS_AND_RISKS.md): decisions already taken, open risks, and cleanup backlog.

Private or client-only requirement files must not be copied here verbatim if
they are marked not-for-GitHub. Summarize the implemented behavior instead.
